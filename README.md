# nurse-records-hub

看護記録の「観察項目固定＋ボタン入力」を起点に、
- 看護計画（選択）
- 観察項目（自動生成）
- 記録（保存：正本）
- 報告書（月次生成）
までを一気通貫で回すための“構造物”。

## 基本原則
- 自由記載は原則禁止（例外は理由付きログ）
- 判断を人から奪わない（王座を作らない）
- 正本はこのシステム側（後段のカルテは提出先・貼り付け先）
- 変更は上書きではなく履歴（Git的運用）

## 構成（ざっくり）
- apps/web: iPad/PCで押すボタンUI（将来：PWA）
- apps/gas: Google Apps Script（Sheets/Drive連携）
- services/core: 観察項目生成・記録ID・履歴管理（中核）
- packages/schemas: データ構造（利用者/計画/観察/記録/報告書）
- packages/reporting: 報告書レンダリング（テンプレ→PDF等）
